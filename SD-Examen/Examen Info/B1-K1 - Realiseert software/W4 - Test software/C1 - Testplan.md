# Info
___
> [!section]- #### Wat is een test plan? ^wat-is-een-testplan
> ___
> Een test plan is een een document waarin je alles omtrent het testen in vastlegt. Denk aan de volgende dingen:
>
> - Wie test.
> - Wat er precies getest wordt.
>   - *Denk aan user stories maar ook bijvoorbeeld aan welke onderdelen van code.*
> - Waar de test in verwerkt/afgerond worden.
> - wanneer een test klaar is .
>   - *Voldoet onder andere aan de acceptatiecriteria.*
> - Hoe er precies getest wordt.


# Minimale Vereisten
___
- Een individuele deelnemer kan uitleggen wat de testprocedure is bij de groep, wat getest is en wat de resultaten zijn.
- Voor slechts 5 user stories zijn tests beschreven.

# Vereisten
___
- Een individuele deelnemer kan uitleggen wat de testprocedure is bij de groep, wat getest is, en wat de resultaten zijn.
- Voor alle user stories zijn tests beschreven.
- In situaties waar gebruikersinvoer verwerkt wordt is goed rekening gehouden met mogelijke edge-cases.
- Bij een technische test zijn meerdere paden beschreven (edge, extreme).
- Het testplan bevat een planning voor de tests. Bijvoorbeeld door dit te verwerken in je definition of done.
- Het is duidelijk in het testplan door wie de test wordt uitgevoerd.
