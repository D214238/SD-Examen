# Info
___
> [!section]- #### Wat is een test rapport? ^wat-is-een-test-rapport
> ___
> Een testrapport is een rapport waarin je een overzicht creëert en reflecteert op het testen. Het is belangrijk dat dit document afgestemd is op de opdrachtgever en dat dit dus niet te technisch is.
> <br>
>
> Het testrapport bevat onder andere:
>
> - Een intro.
> - Een overzicht van wat er allemaal getest is voor de technische tests en voor de acceptatie tests.
>   - Alle tests.
>   - Welke feature en/of user story.
>   - Resultaten.
> - Een overzicht met hoeveel testen en geslaagd en gefaald zijn. Hoe dat verbeterd is en hoe dat dit opgepakt is.
> - Uitleg waarom er getest is wat er getest is.
> - Een analyse. Wat valt op in alle testen.
> - Reflectie op het testen.
>   - Wat ging goed.
>   - Wat ging slecht.
>   - Wat kunnen we er aan doen om het de volgende keer beter te laten gaan.
> - Conclusie.


# Vereisten
___
- Er is een test rapport met resultaten voor alle testscenario’s.
- Er is een conclusie aanwezig.
- Bij enkele resultaten is duidelijk wat de ondernomen (of de nog te ondernemen) acties zijn, of er is gereflecteerd op de manier waarop getest is. er wordt bijvoorbeeld onderbouwt waarom de tests voldoende dekkend waren voor de huidige fase in het ontwikkelingsproces.
- Opbouw van het document is afgestemd op de doelgroep (netheid document voor inzage opdrachtgever).
