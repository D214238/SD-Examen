# Minimale Vereisten
___
- Op enkele benodigde plekken is input validatie op de back-end toegepast.
- Bij enkele SQL queries waar user input gebruikt wordt, wordt juist gebruik gemaakt van prepared statements of een ORM.
- De code laat op veel plaatsen (3 of 4x) herhaling zien.
- De data uit de database wordt in de meeste gevallen niet efficiënt uitgevraagd. (meer dan 2x aantoonbaar door de beoordelaar )

# Vereisten
___
- Op alle benodigde plekken is input validatie op de back-end toegepast.
- Bij alle SQL queries waar user input gebruikt wordt, wordt juist gebruik gemaakt van prepared statements of een ORM.
- De code laat weinig tot geen herhaling zien. Dit wordt opgelost door het maken van functies en/of methoden.
- De data uit de database wordt altijd zo efficiënt mogelijk uitgevraagd.
- Code is volledig schoon, wat blijkt uit:
    - Geen lege/ongebruikte methodes/classes.
    - Geen debug-info die getoond wordt aan gebruikers.
    - Geen onnodige missende witregels tussen methodes, loops, if-statements, etc.
- Er staat nuttige comments in de code
