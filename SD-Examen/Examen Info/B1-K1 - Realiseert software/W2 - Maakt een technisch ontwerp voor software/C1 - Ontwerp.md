# Info
___
> [!section]- #### User stories naar diagrammen ^user-stories-naar-diagrammen
> ___
> Je kan bij de verschillende diagrammen bijvoorbeeld pijltjes trekken en erbij zetten voor welke users story het is, het makkelijkste is door het te noteren als US-1. Op die manier kan je aantonen dat je de meeste user stories verwerkt hebt in diagrammen.


# Minimale Vereisten
___
- Meer dan 50% maar minder dan 80% van de user stories zijn volledig en correct uitgewerkt in het ontwerp.
- Ten minste 1 van de diagrammen is van toegevoegde waarde en beschrijft op een enkele fout na een functionaliteit/user story.

# Vereisten
___
- Alle user stories zijn uitgewerkt in de toepasselijke diagrammen.
- 2 of meer diagrammen zijn van toegevoegde waarde en beschrijven juist een functionaliteit en user story.
    - _Je moet relatie tussen wireframes aangeven, en moet een titel hebben bijvoorbeeld:_ “**WF14 - Product Page**”
    - _Noteer bij voorkeur ook de toebehoren user story._
